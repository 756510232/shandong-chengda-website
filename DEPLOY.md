## 三、如何更新网站内容

改完本地文件后，在 `D:/山东诚达官方网站` 目录下执行：

```bash
git add -A
git commit -m "更新说明（如：修改首页文案）"
git push
```

推送后 GitHub Pages 会在 **1 分钟内** 自动重新构建并上线，无需其他操作。

## 四、更换默认域名（可选进阶）

当前是 `xxx.github.io` 默认域名，不够"企业感"。如需绑定自有域名（如 `chengda.com`）：

1. 购买域名并解析 CNAME 指向 `756510232.github.io`
2. 仓库 **Settings → Pages → Custom domain** 填入你的域名
3. 保存后等待 DNS 生效（几分钟到几小时）

> 注意：若要解析到**国内**服务器，则必须走方案③（国内服务器 + ICP 备案），GitHub Pages 服务器在境外，无法用于国内备案域名。

## 七、接入表单后台（让访客提交进入你的后台）

当前联系表单已写好发送逻辑（`js/main.js` 的 `FORM_ENDPOINT`），但默认是“演示模式”（不真正发送）。
要能在后台看到客户提交的姓名/手机号/车辆信息，需接入一个表单收集服务：

1. 在国内表单平台（如 **金数据 / 表单大师 / 腾讯问卷**）注册并新建一个表单，
   字段建议：姓名、手机号、邮箱、车辆品牌型号、期望额度、留言。
2. 在平台设置里找到“外部提交接口 / Webhook / 表单API”地址并复制。
3. 打开 `js/main.js`，把顶部 `const FORM_ENDPOINT = "";` 的空字符串替换成该地址：
   ```js
   const FORM_ENDPOINT = "https://www.formsmaster.com/f/你的表单ID";
   ```
4. 提交 `git add -A && git commit -m "接入表单后台" && git push`，等待 Pages 重建。

此后访客提交的每条留言都会进入该平台的**后台 / 数据报表**，并可配置邮件通知。
> 选择国内平台是为了合规（车抵贷涉及个人手机号等敏感信息，数据存于国内、不出境）。
